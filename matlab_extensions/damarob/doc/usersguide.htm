Defining a trajectory in operational space:


>> path_des = trajectory(Ts,startpoint,startorient,orient_type,point_1,point_2,...,point_n)


Where:


    Ts: sampling time

    startpoint: three component column vector that specifies the end effector's initial position in operational space

    startorient: three (Euler angles) or four (axis/angle) component column vector that specifies the end effector's initial orientation in operational space

    orient_type: char input ('ZYZ euler', 'ZYX euler', or 'aa') that specifies which representation has been choosed for desired orientation along the trajectory

    point_1, point_2... : struct inputs (as many as you like) containing information on how the path has to be generated.

    Struct inputs contain:

Rectilinear path
input
|
+--g % goal point (column vector)
+--tf % time to reach goal point (seconds)
+--path_type = 'r' % specifies the type of path (rectilinear)
+--via % point is via point (1) or not (0)
+--orient % this can be a three component vector (for ZYX or ZYZ Euler angles)
or a four component one (for axis/angle representation).


Circular path
input
|
+--g % goal point (column vector)
+--tf % time to reach goal point (seconds)
+--path_type = 'c' % specifies the type of path (circular)
+--thf % radians to be sweeped by the position vector
+--c % centre of the circle
+--z % axis of rotation
+--via % point is via point (1) or not (0)
+--orient % this can be a three component vector (for ZYX or ZYZ Euler angles)
or a four component one (for axis/angle representation).



manipulator Class:


A manipulator object contains the following internal structure:


Manipulator
object
|
+--name % string: manipulator name
|
+--DH % struct containing: table,r,p,z
| +--table % DH table in a matrix with RP flag and limits
| +--r % number of rows of DH table (number of links)
| +--p
| | +--p0 % origin of 0-frame in b-frame
| | +--pe % origin of e-frame in b-frame
| | +--p{1}..p{n} % origin of i-frame in b-frame
| +--z
| | +--z0 % z-axis of 0-frame in b-frame
| | +--ze % z-axis of e-frame in b-frame
| | +--z{1}..z{n} % z-axis of i-frame in b-frame
|
+--jointvar % struct (syms): q qd qdd: joint variables
| +--q % symbolic variable for joint position
| +--qd % symbolic variable for joint velocities
| +--qdd % symbolic variable for joint accelerations
|
+--kin % struct containing kinematics data
| |
| +--T % homogeneous transformation matrices
| | +--Tb0 % homogeneous transformation matrices from b-frame to 0-frame
| | +--Tne % homogeneous transformation matrices from n-frame to e-frame
| | +--Tbe % homogeneous transformation matrices from b-frame to e-frame
| | +--T{1}..T{n} % homogeneous transformation matrices from 0-frame to i-frame
| |
| +--J % geometric Jacobian matrix
| | +--J % geometric Jacobian matrix
| | +--Jd % time-derivative of geometric Jacobian matrix
| | +--Jp_l{1}..Jp_l{n} % geometric Jacobian matrix [joints' COM position]
| | +--Jo_l{1}..Jo_l{n} % geometric Jacobian matrix [joints' COM orientation]
| | +--Jp_m{1}..Jp_m{n} % geometric Jacobian matrix [rotors' COM position]
| | +--Jo_m{1}..Jo_m{n} % geometric Jacobian matrix [rotors' COM orientation]
| |
| +--nsap % orientation-position matrix in a column: [n;s;a;p]
| |
| +--R
| | +--R{1}..R{n} % rotation matrices from 0-frame to i-frame
| |
| +--A
| +--A{1}..A{r} % homogeneous transformation matrices from (i-1)-frame to i-frame
|
+--dyn % struct containing dynamics data
| +--B % manipulator inertia matrix
| +--C % Coriolis and centripetal matrix (Christoffeln signs conventions)
| +--g % gravity column vector
| +--data % input data
| +--g0 [gx; gy; gz] % gravity in 0-frame
| +--ml [1 .. r] % links mass
| +--mm [1 .. r] % rotors mass
| +--kr [1 .. r] % motorgear ratio
| +--zm {1 .. r} % motor axes expressed in [i-frame]; zm{i} is a 3x1 column vector
| +--Il {1 .. r} % links' mass moment of inertia [i-frame]; Il{i} is a 3x3 matrix
| +--Im {1 .. r} % motors' mass moment of inertia; Im{i} is a 3x3 matrix
| +--Fv [rxr] % joints' viscous friction
| +--Fm [rxr] % motors' viscous friction
| +--pl {1 .. r} % position of centres of mass of the
| | i-link with %respect to i-frame; pl{i} is a 3x1 column vector
| +--pm {1 .. r} % position of motors with respect to the
| previous joint %axes; pm{i} is a 3x1 column vector
|
+--dyndata % flag: 1 means dynamic dataset ready
|
+--evaluated % struct: contains flags
| +--Direct_Kinematics % 1 means Direct_Kinematics evaluated
| +--Direct_Dynamics % 1 means Direct_Dynamics evaluated
|
+--func % struct: handles of all m-functions
| +--x % direct kinematics function: x position
| +--y % direct kinematics function: y position
| +--z % direct kinematics function: z position
| +--Jd_qd % product: J' * q'
| +--Jt_he % product: J^T * he
| +--J_qdd % product: J * q''
| +--nsap % full direct kinematics function: nsap in a column
| +--B_matrix % inertia matrix in a column
| +--B_qdd % product: B * q''
| +--C_qd % product: C * q'
| +--g % gravity vector
|
+--vr_base % vrworld: manipulator's wrl vr-object
+--fig % struct: handles of all figures


A manipulator object is associated with following files:


    MANIP_NAME_3D.png

    MANIP_NAME_B_matrix.m

    MANIP_NAME_B_qdd.m

    MANIP_NAME_C_qd.m

    MANIP_NAME_g.m

    MANIP_NAME_Jd_qd.m

    MANIP_NAME_J_qdd.m

    MANIP_NAME_Jt_he.m

    MANIP_NAME_nsap.m

    MANIP_NAMEVR_base.wrl


These are m-function files, a VRML model and a PNG snapshot of the modelled manipulator.



Defining a Manipulator:


Use manipulator constructor to create manipulator objects:


>> manip = manipulator(DH,name,Tb0,Tne)


Where:


    manip: is the instance created

    name: is a string containing the description name of the robot

    DH: Denavit-Hartenberg table, including joint type and limits.
    DH is a (r x 7) matrix as follows: [a alpha d theta R/P min max]
    Each row contains a link data.
    R/P is a flag variable:
    R/P = 0 indicates a revolute joint
    R/P != 0 indicates a prismatic joint
    if no limits (min - max) is specified, range will be [-Inf, Inf]

    Tb0: Homogeneous transformation matrix between b-frame and 0-frame

    Tne: Homogeneous transformation matrix between n-frame and e-frame
    If not specified, Tb0 and Tne will be assumed identities



Inserting dynamics data:


Use InsertDyndata method to insert dynamics data into a manipulator object:


>> manipulator.InsertDyndata(dyndata)


Where dyndata is a struct containing all these fields:

dyndata
|
+--g0 [gx; gy; gz] %gravity in 0-frame
+--ml [1 .. r] % links' masses
+--mm [1 .. r] % rotors' masses
+--kr [1 .. r] % motorgears' ratios
+--zm {1 .. r} % motor axes expressed in [i-frame]
+--Il {1 .. r} % links' mass moments of inertia [i-frame]
+--Im {1 .. r} % motors' mass moments of inertia
+--Fv [rxr] % joints' viscous frictions
+--Fm [rxr] % motors' viscous frictions
+--pl {1 .. r} % position of center of mass of the
| % i-link with respect to i-frame
+--pm {1 .. r} % position of motors with respect to the
% previous joint axes

Please note:
Il{i} is a 3x3 matrix
Im{i} is a 3x3 matrix
pl{i} is a 3x1 column vector
pm{i} is a 3x1 column vector
zm{i} is a 3x1 column vector

Saving Manipulator:


Use standard 'save' command to save manipulator objects:


>> save FILENAME % save all workspace variables in file FILENAME

>> save FILENAME M1 M2 % save variable 'M1' and 'M2' in file FILENAME, also if one

% or more of them are manipulator objects.



Loading Manipulator:


Use manipload function to load manipulator objects from file. This will automatically generate files associated with manipulator without evaluating specific functions:


>> manipload FILENAME % load all variables stored in FILENAME

>> manipload FILENAME x y z % load only x y z from FILENAME



Concatenating Manipulators:


Use + operator (plus method) to concatenate manipulator objects.
This will automatically generate a new manipulator with:

    Tb0 of the first manipulator

    all links of the first manipulator

    all links of the second manipulator

    Tne of the second manipulator

>> manipC = manipA + manipB

Note that manipA and manipB must be both numeric or symbolic.

Remember that:

    you can create void manipulators containing just Tb0 and Tne that you can use to modify these homogeneous transformations of concatenated manipulators.

    you can rename manipulators with rename method
